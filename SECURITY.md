# Security Policy

## Repository Purpose
Public-facing documentation of workflows and outcomes, not sensitive configuration storage.

## Data Handling Rules
- No real private addressing
- No credentials, tokens, secrets, or private keys
- No internal host fingerprints or sensitive backup topology

## Vulnerability Reporting
Report potential exposures privately via GitHub Security Advisories or direct maintainer contact.

## Response Standard
Potential sensitive-data exposure is treated as urgent and remediated immediately.
